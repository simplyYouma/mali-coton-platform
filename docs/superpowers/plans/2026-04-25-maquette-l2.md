# Maquette Interactive L2 — Plan d'implémentation

> **Pour les workers agentiques** : utiliser `superpowers:executing-plans` pour exécuter ce plan tâche par tâche. Les étapes utilisent la syntaxe checkbox `- [ ]`.

**Goal** : Livrer une maquette interactive React+TS, navigable, conforme L2 (CDC §6.2) — parcours complets des 3 profils (agent / superviseur / admin), Phase 1 + aperçus Phase 2, app tablette offline simulée, bilingue FR/BM, design system appliqué.

**Architecture** : SPA React 18 + TS strict, feature-based, MSW pour mocker l'API, IndexedDB+Outbox pour offline tablette, tokens CSS centralisés, composants 100% custom (Yumi Standard).

**Tech Stack** : React 18, TypeScript strict, Vite, React Router v6, TanStack Query, Zustand, React Hook Form + Zod, Chart.js, Leaflet, i18next, MSW, Dexie, Vitest + Playwright, Storybook 8.

**Références obligatoires** :
- [docs/design-system.md](../../design-system.md) — tokens, composants, règles A11y
- [docs/tech-spec.md](../../tech-spec.md) — architecture, contrats API, offline-first
